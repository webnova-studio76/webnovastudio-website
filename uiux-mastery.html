Phase 01 // UX Research Basics
Level: Beginner • Study Time: 2 Hours

Design Thinking & Discovery

UX Research helps us understand what target users truly need. Design Thinking ensures we fix actual user frustrations instead of guessing what might look nice.

📖 Learn & Understand

User Personas & Empathy Mapping

A User Persona is a profile card representing your average target user. To create one, we run interviews to document a user's clear daily goals, technical background, and major frustrations. This maps directly into an Empathy Map, which splits user field research variables into four simple categories: What users Say, Think, Do, and Feel.

#UserResearch#EmpathyMapping#DesignThinking
📖 Learn & Understand

Problem Statements & User Journeys

Great designs start with a clear problem definition. We frame these as "How Might We" statements to guide ideation loops. This flows straight into a User Journey Map, which visually tracks every single step a user takes to hit a goal inside an application—recording their actions and emotional drops along the way.

Mini-Project Milestone Assignment

Task Challenge: Map out 2 explicit User Personas and an Empathy Profile layout for a local grocery delivery app. Document three primary customer pain points regarding checkout delays.

Phase 02 // Information Architecture
Level: Beginner • Study Time: 3 Hours

Wireframing & Sitemaps

Information Architecture means organizing data clearly so app users never get lost finding navigation links.

📖 Learn & Understand

Low-Fidelity Sketching & Navigation Logic

Before jumping into digital design tools, engineers use low-fidelity paper sketching to quickly map out screen content distributions. This maps structural navigation layouts without getting distracted by custom brand colors or image assets.

#Sitemaps#Wireframing#LayoutGrids
The Power of the 8pt Grid System

Modern UI layout relies completely on structural layout scales built around multiples of 8 pixels (8, 16, 24, 32, etc.). Using an 8pt layout grid ensures that spacings, paddings, and line heights scale cleanly across varying screen sizes without broken fractions.

Phase 03 // Visual UI Properties
Level: Beginner • Study Time: 4 Hours

Color Theory & Typographic Hierarchies

Visual design blends aesthetic character blocks with mathematical accessibility standards to make screens clean and highly readable.

Color Psychology & Accessibility Limits

Colors steer how users feel inside an interface. For example, secure banking interfaces use deep royal blues to build user trust, while medical systems prefer soft greens. According to global WCAG accessibility rules, text contrast ratios must hit at least a 4.5:1 scale against background cards so visually impaired users can read them clearly.

#ColorTheory#Typography#Accessibility
Font Category Visual Structural Character Best Use Case Fit
Serif (e.g., Times New Roman) Features small decorative feet or strokes on the tips of letters. Editorial prints, heavy text blocks, traditional brand styleguides.
Sans-Serif (e.g., Inter, Helvetica) Clean, geometric letterforms with completely square, modern cuts. Digital user interfaces, mobile screen applications, terminal outputs.
Phase 04 // Scalable UI Engineering
Level: Intermediate • Study Time: 5 Hours

Atomic Design Systems

A **Design System** is a centralized library of reusable code components and visual patterns that ensures massive applications remain uniform across production branches.

Atomic Design Principles

We build scalable interface assets by following the Atomic Design methodology. This structure scales interface elements across five clear layers:

  • Atoms: Individual, un-splittable UI elements (such as a single label, input field, or icon).
  • Molecules: Combinations of atoms bonded together to form a simple functional component (like pairing an input atom with a button atom to build a Search Box).
  • Organisms: Complex components formed by grouping multiple molecules together (such as a website Header navigation bar).
#AtomicDesign#AutoLayout#ComponentLibraries
⚠ Common Student Design Mistake

Do not draw buttons as static vector rectangles. Always structure interface cards and buttons using Auto-Layout dynamic wrappers. This ensures text paddings scale dynamically when your content string changes across translated layouts.

Phase 05 // Motion Logic Matrices
Level: Intermediate • Study Time: 6 Hours

High-Fidelity Prototyping & Animation

Prototypes simulate active application actions, letting design teams test interactions on real users before developers write any code.

Micro-interactions & Smart Transitions

Micro-interactions provide clear visual feedback for user actions—like a button subtly lifting when hovered over, or a loading wheel spinning during a form submission. Using motion tools like Figma's Smart Animate lets you build high-fidelity interactive prototypes that look and feel just like real code environments.

#MicroInteractions#SmartAnimate#UserTesting
Phase 06 // Production Deployment Tracks
Level: Intermediate • Study Time: 4 Hours

Design-to-Code Engineering Handover

The final stage translates completed screen layouts into raw developer assets, transforming layouts into functional production code fields.

Inspecting Styles for clean CSS Translation

To ensure pixel-perfect conversion, designers document design variables as standardized Design Tokens. These tokens bridge design files and frontend production by turning visual properties (like borders, drop shadows, and font values metrics) into clear, reusable CSS variable definitions.

#DesignTokens#ResponsiveCSS#HandoffSpecs
