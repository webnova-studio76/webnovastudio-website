Virtualization: The Cloud Engine

Virtualization is the technology that allows us to run multiple "virtual" computers on a single piece of physical hardware.

The Hypervisor

The software that creates and runs Virtual Machines (VMs). It sits between the physical hardware and the VMs.

  • Type 1 (Bare Metal): Runs directly on the hardware. Ex: VMware ESXi, Microsoft Hyper-V. Used in major data centers.
  • Type 2 (Hosted): Runs as an app on top of an OS. Ex: Oracle VirtualBox, VMware Workstation. Great for personal testing.

Virtual Machine Management

In the cloud, you don't buy servers; you launch instances or virtual machines.

AWS EC2

Elastic Compute Cloud. AWS's flagship compute service. Highly customizable with "Instance Types" for memory, compute, or GPU needs.

Azure Virtual Machines

Microsoft's scalable VM service. Excellent integration for Windows-heavy environments and SQL Server workloads.

Scaling Mechanisms:

  • Vertical Scaling (Up): Making a single server "bigger" (more RAM/CPU).
  • Horizontal Scaling (Out): Adding more servers to handle the load.

Serverless Computing

Serverless doesn't mean there are no servers; it means you don't manage them. You only provide the code.

Function as a Service (FaaS)

  • AWS Lambda: Runs code in response to events (like a file upload or a web request).
  • Azure Functions: Similar event-driven compute that integrates with Microsoft's cloud ecosystem.
The Pay-Per-Request Model: With VMs, you pay for every hour the server is "on." With Serverless, you only pay for the exact milliseconds your code is running. If no one visits your site, you pay $0!
