🛒 Capstone: E-Commerce DB Engine

Design and implement a relational database for a modern digital storefront.

The Blueprint

Before writing code, we must define our entities and their relationships. This project requires 3rd Normal Form (3NF) to avoid data anomalies.

Entities & Relationships:

  • Users: (PK: user_id) - Stores profile data.
  • Products: (PK: prod_id) - Stores inventory & prices.
  • Orders: (PK: order_id, FK: user_id) - Links users to their purchases.
  • Order_Items: (Composite PK, FKs) - A join table resolving the M:N relationship between Orders and Products.

Building the Schema

Use these DDL commands to set up the relational structure with proper Foreign Key constraints.

-- 1. Create Products Table
CREATE TABLE Products (
    p_id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    price DECIMAL(10, 2),
    stock INT DEFAULT 0
);

-- 2. Create Orders Table
CREATE TABLE Orders (
    o_id INT PRIMARY KEY AUTO_INCREMENT,
    u_id INT,
    order_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (u_id) REFERENCES Users(u_id)
);

-- 3. Create Order_Items (Bridge Table)
CREATE TABLE Order_Items (
    o_id INT,
    p_id INT,
    quantity INT,
    PRIMARY KEY (o_id, p_id),
    FOREIGN KEY (o_id) REFERENCES Orders(o_id),
    FOREIGN KEY (p_id) REFERENCES Products(p_id)
);

Business Intelligence Queries

A database is only useful if you can extract insights. Try these complex queries:

Query 1: Total Revenue per Product

SELECT p.name, SUM(oi.quantity * p.price) AS revenue
FROM Products p
JOIN Order_Items oi ON p.p_id = oi.p_id
GROUP BY p.name
ORDER BY revenue DESC;
