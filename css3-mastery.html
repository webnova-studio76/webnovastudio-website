CSS Document Foundations

Cascading Style Sheets (CSS) determine the presentation of web pages. They map cosmetic properties over raw HTML bones by calculating rule priorities across cascade branches.

📖 Learn & Understand

Selectors, Specificity, and the Cascade

When multiple contradictory styles target a single HTML element, the browser runs the Cascade rule to pick a winner. It measures Specificity using an explicit hierarchy score weight: Inline style attributes win completely, followed by ID selectors (weight: 100), Class attributes (weight: 10), and raw Element tags (weight: 1). If scores tie, the latest rule declared at the bottom of the style sheet wins automatically.

Phase 02 // Element Spatial Geometry
Level: Beginner • Study Time: 2 Hours

The Core Box Model

Every element inside a web document browser window behaves as a separate rectangular box layout block wrapping content spaces boundaries.

Box-Sizing Formulas: Content-Box vs. Border-Box

By default, browser layouts compute sizes using content-box, adding padding and borders *on top* of width declarations, often causing element sizes to burst container boxes. Modern interface engineers override this globally using box-sizing: border-box. This formula folds borders and paddings *inside* your declared dimensions, keeping element sizes perfectly accurate.

#BoxModel#BorderBox#DisplayRules
📐 Layered Structural Breakdown of Box Model Dimensions

Margin (Outer Space) → Border (Outline Boundary) → Padding (Inner Breathing Room) → Content Core

Display Property Line Break Generation Width / Height Allotments
display: block Forces an automatic line break, starting a fresh row. Claims 100% container width natively, respects manual sizes.
display: inline Sits inside line strings without forcing any breaks. Ignores width/height codes completely, wraps around content sizes.
display: inline-block Sits inside string rows smoothly without breaks. Honors manual structural sizing dimensions perfectly.
Phase 03 // 1D Fluid Alignment Interfaces
Level: Intermediate • Study Time: 4 Hours

Modern Flexbox Layout Engines

Flexbox acts as a one-dimensional fluid layout engine designed to distribute elements accurately across either a singular row or column track lane.

Justify-Content vs. Align-Items Alignment Rules

Flex arrangements manipulate layouts along alternative axis lines. justify-content governs item packaging along the main flow axis (e.g., packing elements to the left, right, center, or spacing them out uniformly). align-items governs layout balancing orthogonally across the cross axis row (e.g., centering item content heights vertically inside a global navigation header).

#Flexbox#AxisAlignment#FluidLayouts
Flex-Basis, Grow, and Shrink Parameters

Use the combined flex shorthand (e.g., flex: 1 0 200px) to control fluid items. This maps item behavior: Grow (1 allows items to expand to claim leftover space), Shrink (0 blocks items from squeezing down below boundaries), and Basis (sets the fallback structural starter width size).

Phase 04 // 2D Grid Design Platforms
Level: Intermediate • Study Time: 4 Hours

CSS Grid Layout Architecture

CSS Grid provides a powerful, two-dimensional layout matrix engineered to arrange content rows and columns concurrently.

Fractional Units (fr) & Grid Template Frameworks

Grid environments introduce the fractional unit (fr), which represents a fluid slice of leftover layout space. Declaring grid-template-columns: repeat(3, 1fr) slices a container canvas into three perfectly equal columns positions. This pairs with grid-gap parameters to separate grid cards without traditional margin collapse blocks bugs.

Phase 05 // Device Optimization Frameworks
Level: Beginner • Study Time: 3 Hours

Responsive Web Adaptation

Responsive development uses media query conditional hooks to alter visual parameters smoothly across different devices.

Relative Unit Architecture: rem and em metrics

To avoid rigid, broken pixel structures on small mobile screens, scalable platforms utilize fluid relative sizing metrics. The rem unit computes sizes proportionally from the root font properties of the document canvas (usually 16px). The em unit calculates values recursively from its immediate parent container element, making card paddings scale automatically with font variations.

⚠ Common Student Responsive Design Mistake

Never start coding layouts from large desktop monitor sizes using high max-width parameters. Always implement a Mobile-First design loop, establishing clean baseline rules for small viewports first, then layering up expansions via min-width media queries.

Syllabus Question Bank

2 Marks // Standard Core Metric Q: Explain the explicit layout behavior achieved via the rule box-sizing: border-box.

A: The box-sizing: border-box rule forces the web browser to compute an element's total layout width and height by including padding and borders *inside* the declared dimensions. This prevents cells from expanding or bursting container boundaries when inner spatial margins change.

5 Marks // Frequently Asked Paper Question Q: Contrast the architectural layout design goals separating CSS Flexbox from CSS Grid engines.

A: CSS Flexbox is a one-dimensional fluid layout engine designed to arrange items sequentially along a single axis lane (either a horizontal row or a vertical column). CSS Grid is a two-dimensional layout system engineered to manage both alignment rows and column card frameworks simultaneously across a structured grid matrix.
