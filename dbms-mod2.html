SQL Language Categories

SQL is divided into sub-languages based on their function within the database.

Category Full Name Commands Purpose
DDL Data Definition CREATE, ALTER, DROP Defining table structures.
DML Data Manipulation INSERT, UPDATE, DELETE Managing data within tables.
DQL Data Query SELECT Retrieving data.
DCL Data Control GRANT, REVOKE Permissions and Security.
-- Example DDL: Creating a table
CREATE TABLE Students (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    grade CHAR(1)
);

Mastering Joins

Joins are used to combine rows from two or more tables based on a related column.

  • Inner Join: Returns records that have matching values in both tables.
  • Left Join: Returns all records from the left table, and matched records from the right.
  • Right Join: Returns all records from the right table, and matched records from the left.
  • Full Outer Join: Returns all records when there is a match in either left or right table.
SELECT Orders.id, Customers.name
FROM Orders
INNER JOIN Customers ON Orders.customer_id = Customers.id;

Aggregate Functions & Subqueries

1. Aggregate Functions

These perform a calculation on a set of values and return a single value.

  • COUNT(): Returns the number of rows.
  • SUM(): Returns the total sum of a numeric column.
  • AVG(): Returns the average value.
  • MAX() / MIN(): Returns the largest/smallest value.

2. Subqueries

A "Query within a Query." Used to perform complex filtering.

-- Finding students with grades above average
SELECT name FROM Students
WHERE score > (SELECT AVG(score) FROM Students);
GROUP BY: When using aggregate functions, you often need the GROUP BY statement to group results by one or more columns (e.g., total sales per city).
