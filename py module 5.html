Classes, Objects & Constructors

A Class is a blueprint for creating objects. An Object is a specific instance of that class.

The Constructor: __init__

This special method initializes the object's attributes when it is created.

class Car:
    def __init__(self, brand, model):
        self.brand = brand  # Attribute
        self.model = model

# Creating an Object (Instance)
my_car = Car("Tesla", "Model S")
print(my_car.brand) # Output: Tesla
What is 'self'? It represents the specific instance of the object being created. It allows the class to distinguish between different objects (e.g., your car vs. my car).

Inheritance & Polymorphism

1. Inheritance

Allows a "Child" class to inherit attributes and methods from a "Parent" class, promoting code reuse.

class Animal:
    def speak(self):
        print("Animal makes a sound")

class Dog(Animal): # Dog inherits from Animal
    def speak(self):
        print("Woof! Woof!")

2. Polymorphism

The ability for different classes to be treated as instances of the same parent class through the same interface (like the speak() method above).

Encapsulation & Data Hiding

Encapsulation involves wrapping data (attributes) and methods into a single unit and restricting direct access to some components.

Private Attributes

In Python, we use a double underscore (__) to make an attribute private, preventing it from being modified directly from outside the class.

class BankAccount:
    def __init__(self, balance):
        self.__balance = balance # Private attribute

    def get_balance(self):
        return self.__balance

account = BankAccount(1000)
# print(account.__balance) # This would cause an ERROR
Why hide data? It prevents accidental corruption of important data and allows the class to control how its information is updated (e.g., ensuring a bank balance never goes below zero).
