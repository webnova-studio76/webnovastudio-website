Identity & Access Management (IAM)

IAM is the framework that ensures the right people have the right access to the right resources.

The Three Pillars of IAM:

  • Users: Specific individuals (like you).
  • Groups: Collections of users (e.g., "Developers," "Admins").
  • Roles: Temporary identities that can be assumed by users or services (e.g., allowing an EC2 instance to read from an S3 bucket).
  • Policies: JSON documents that define what actions are allowed or denied.
The Principle of Least Privilege: Always grant the minimum amount of access required to perform a task. Never use the "Root" account for daily work!

Infrastructure as Code (IaC)

Instead of clicking buttons in the cloud console, we write code to define our infrastructure. This makes building 100 servers as easy as building one.

Popular Tools:

  • Terraform (HashiCorp): Open-source and cloud-agnostic (works with AWS, Azure, and Google Cloud).
  • AWS CloudFormation: AWS's native tool for managing resources.
# Example Terraform Snippet
resource "aws_instance" "web_server" {
  ami = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
}

CI/CD Pipelines

Continuous Integration (CI) and Continuous Deployment (CD) automate the process of testing and shipping code to the cloud.

The Lifecycle:

  1. Commit: Developer pushes code to GitHub.
  2. Build: The system compiles the code and builds a Docker container.
  3. Test: Automated tests run to ensure there are no bugs.
  4. Deploy: The code is automatically pushed to the cloud servers.
DevOps Culture: DevOps isn't just a tool; it's a culture of collaboration between Developers (Dev) and Operations (Ops) to ship software faster and more reliably.
