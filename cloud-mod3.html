Object vs. Block Storage

Choosing the right storage type depends on how you plan to access the data.

Feature Object Storage (S3 / Blob) Block Storage (EBS / Managed Disk)
Best For Photos, Videos, Static Files, Backups Operating Systems, Databases (high I/O)
Access Via HTTP/API (Web) Attached directly to a VM
Scaling Virtually Infinite Limited to the size of the volume
S3 (Simple Storage Service): In AWS, S3 is used to host millions of files. It is highly durable (99.999999999% durability), meaning you will almost never lose a file stored there.

Virtual Private Clouds (VPC)

A VPC is your private corner of the public cloud. It’s a logically isolated network where you launch your resources.

Key Components:

  • Subnets: Slicing your VPC into smaller networks.
    • Public Subnet: Accessible from the internet (e.g., your Web Server).
    • Private Subnet: Not accessible from the internet (e.g., your Database).
  • Internet Gateway (IGW): The "door" that allows traffic into your public subnet.
  • Security Groups: Virtual firewalls that control traffic at the instance level.

Global Performance & Availability

1. Content Delivery Networks (CDN)

A CDN (like AWS CloudFront) caches your content in "Edge Locations" around the world. If a user in India visits your US-based site, they get the images from a server in India, making it much faster.

2. Load Balancing

A Load Balancer acts as a "traffic cop." It distributes incoming web requests across multiple servers so that no single server gets overwhelmed.

Health Checks: Load Balancers perform "Health Checks." If one of your servers crashes, the Load Balancer stops sending traffic to it automatically until it is fixed.
