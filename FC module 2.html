CPU & Processing Logic

The Central Processing Unit (CPU) follows the Von Neumann Architecture, where instructions and data share the same memory space.

The Fetch-Decode-Execute Cycle

  • Fetch: CU retrieves instructions from RAM.
  • Decode: Instructions are translated into binary commands.
  • Execute: ALU performs the mathematical or logical task.
  • Store: Results are written back to registers or memory.

Pipelining

In modern architecture, we use Pipelining to process multiple instructions at different stages simultaneously, acting like an assembly line.

Pro Tip: Higher Clock Speed (GHz) means more cycles per second, but architecture efficiency (IPC) is often more important for real speed!

Memory Hierarchy & Speed

Component Volatility Role
Registers Volatile Immediate CPU data (Ultra Fast)
Cache (L1-L3) Volatile Frequent data buffer between CPU/RAM
RAM Volatile Active programs and OS data
NVMe SSD Non-Volatile Permanent file and OS storage

Virtual Memory

When RAM is full, the OS uses a portion of the SSD as "Virtual Memory." This prevents crashes but significantly slows down performance because SSDs are slower than RAM.

The Physical Infrastructure

The Bus System

  • Data Bus: Moves actual content between components.
  • Address Bus: Specifies the memory location for data.
  • Control Bus: Sends command signals (Read/Write).

Thermal Management & Throttling

Processing billions of bits creates friction and heat. To protect hardware, systems use:

  • Thermal Throttling: Slowing the clock speed if heat limits are hit.
  • Heatsinks & Paste: Pulling heat away from the silicon chip.
  • Active Cooling: Fans or Liquid Cooling systems.

Modern Trends in 2026

CPU vs GPU (Parallelism)

CPU: Few powerful cores for sequential logic.
GPU: Thousands of simple cores for parallel tasks like AI and 3D rendering.

Instruction Sets: x86 vs ARM

  • x86 (Intel/AMD): High performance, complex (CISC), found in workstations.
  • ARM (Apple/Snapdragon): Reduced Instruction Set (RISC), extremely power-efficient for mobile.

Secondary Storage

NVMe M.2 SSDs plug directly into PCIe lanes, making them up to 10x faster than traditional SATA SSDs.
