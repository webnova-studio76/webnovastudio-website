01 // Threats & Defense Mechanisms
Difficulty: Medium • Study Time: 12 Mins

Threats & Defensive Measures

Cybersecurity is the core practice of protecting digital hardware systems, network transmission lines, and user data fields from unauthorized exploits and malicious attacks.

The Five Core Categories of Malware

Malware is simply any malicious code designed to disrupt, damage, or gain unauthorized entry to a machine container.

🦠 Virus

A malicious script that attaches itself to an innocent host file. It spreads to other files only when a user manually interacts with or executes that host file.

🪱 Worm

A highly dangerous, standalone piece of self-replicating code. It does not need human help to trigger; it crawls automatically through shared networks to infect other machines.

🔒 Ransomware

Critical Exam Focus

A malicious attack that completely encrypts your local files, locking you out of your data completely until a financial ransom is paid to the attacker.

Spyware & Trojans

Disguises itself as legitimate software to trick users into running it, while silently logging keystrokes or tracking data inputs behind the scenes.

Attacker Spawns Fake Email/Link → User Clicks → Enters Real Password → Attacker Steals Credentials Buffer

Core System Defense Infrastructure Layers

  • Network Firewalls: Barrier systems that block unauthorized external network connections based on pre-set security rules.
  • Cryptographic Encryption: Scrambles readable data into unreadable text strings, ensuring files remain useless even if intercepted by bad actors.
  • Multi-Factor Authentication (MFA): Adds an extra layer of identity verification (like an OTP sent to your phone) beyond a basic username and password.
02 // Sizing & Distributed Architectures
Difficulty: Easy • Study Time: 10 Mins

File Systems & Virtualization

Cloud computing moves local processing and storage tasks off individual devices and pushes them onto secure, distributed server warehouses accessed over the internet.

The Three Core Cloud Service Models

Cloud offerings are organized into three standard service layers based on how much configuration control the client retains:

Cloud Service Model What You Control & Manage What the Provider Manages 🖥️ Real-World Example
IaaS (Infrastructure) Operating System, Custom Apps, Data Raw physical servers, storage, networking Amazon Web Services (AWS), Microsoft Azure
PaaS (Platform) Your own application source code Operating System, Server runtime engine, Hardware Google App Engine, Heroku workspace slots
SaaS (Software) Almost nothing (Just use the app) Everything (Code, OS, Servers, Storage) Gmail, Netflix, Canva web instances
Elastic Scalability Advantage

The core commercial power of cloud computing lies in **Elastic Scalability**. If your web traffic suddenly spikes due to a marketing event, your platform can scale up its RAM and server counts in milliseconds, scaling them back down automatically when traffic drops to save costs.

03 // Data Mining & Machine Learning Mechanics
Difficulty: Medium • Study Time: 9 Mins

Artificial Intelligence & Privacy

Artificial Intelligence (AI) focuses on engineering software loops that can simulate human logical choices, recognize patterns, and learn from experience.

🧠 The Machine Learning Data Training Pipeline [Diagram flowchart mapping the processing route from raw datasets inputs into an AI prediction model]

Raw Dataset Inputs → Training Algorithm Analysis → Pattern Model Extraction → Live Prediction Output

AI in Daily Life (Systems You Already Use)

  • Content Recommendations: Platforms like YouTube and Instagram analyze what you watch to instantly suggest matching videos.
  • Smart Navigation: Google Maps processes live location telemetry from millions of smartphones to calculate transit delays and traffic conditions ahead.

Digital Privacy Challenges in the AI Era

Because modern AI systems require massive pools of data to train effectively, they present significant new privacy risks across public sectors:

  • Data Mining Traps: Online platforms often scrape private text posts, search queries, and location histories to train pattern matching models.
  • Deepfake Malicious Code: AI model generations can synthesize realistic video and voice clones of real individuals, creating a powerful vector for misinformation.
  • Anonymization Constraints: Scraping data requires removing personal identifiers, but advanced AI algorithms can reverse this process by cross-referencing datasets.
Core AI Hierarchy Memory Trick

Keep the subset layers of computer intelligence straight for exams using this simple chart: Artificial Intelligence is the broad universe → Machine Learning is the inner subset that learns from data patterns → Deep Learning is the core kernel using multi-layer neural networks to mimic brain cells.

2 Marks // Frequently Asked Q: State the core difference separating a Computer Virus from a Worm module.

A: A Virus requires human help to trigger and spread because it must attach itself to an innocent host file, executing only when a user opens that file. A Worm is a standalone piece of self-replicating malware that spreads automatically across shared networks without any human interaction.

5 Marks // Core Syllabus Metric Q: Contrast the infrastructure provisions defining IaaS vs. SaaS cloud models.

A: Infrastructure as a Service (IaaS) provides raw, unconfigured virtual hardware and servers (like AWS), leaving the client responsible for installing the operating system, middleware, and applications. Software as a Service (SaaS) delivers a fully finished application ready to use inside a browser window (like Gmail), leaving all maintenance entirely to the service provider.

Laboratory Viva Questions Checklist

  • What is the purpose of Two-Factor Authentication (2FA)? It adds a second layer of identity verification (like an SMS token) to protect accounts even if an attacker steals your password.
  • Why do machine learning algorithms require huge pools of user data? Because training algorithms extract accurate patterns and predictions based on the scale and diversity of the input records.
  • What does the abbreviation GIGO highlight during computing audits? Garbage In, Garbage Out; proving that if you feed broken input logic into a system, it will inevitably generate broken output results.

One-Minute Summary Revision Sheet
