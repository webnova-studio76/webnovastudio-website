Software Classifications

System Software

The bridge between hardware and user.

  • Operating Systems (Windows, Linux)
  • Device Drivers
  • Utilities (Antivirus, Compressors)

Application Software

Designed for specific end-user tasks.

  • General Purpose (Browsers, Office)
  • Bespoke/Custom (Banking Apps)
GUI vs CLI: Most users prefer Graphical Interfaces (clicking), but professionals use Command Line Interfaces (text) for speed and automation.

The Kernel & Mode Management

The Kernel is the core program of the OS that manages all hardware interactions.

Execution Modes

  • User Mode: Restricted access for apps to prevent crashes.
  • Kernel Mode: Full access for the OS to manage memory and CPU.

Advanced Memory Management

  • Paging: Breaking programs into small blocks for efficiency.
  • Virtual Memory: Using SSD space as "emergency RAM" when physical RAM is full.

File Systems & Virtualization

System Platform Key Feature
NTFS Windows Security permissions & encryption.
APFS macOS Optimized for Flash/SSD storage.
FAT32 Universal Compatible with everything (4GB limit).
EXT4 Linux High performance and stability.

Virtualization

Using a Hypervisor (like VirtualBox), you can run multiple Operating Systems on a single physical machine. This is the foundation of Cloud Computing.

The Booting Process (POST to GUI)

The sequence of starting a computer is called "Booting."

The BIOS/UEFI

A firmware program on the motherboard that initializes hardware.

Step-by-Step Sequence:

  • 1. POST: Power-On Self-Test checks for hardware health.
  • 2. Bootstrap Loader: Locates the OS on the SSD.
  • 3. Kernel Load: The heart of the OS is moved into RAM.
  • 4. Initialization: Drivers and GUI are launched for the user.
